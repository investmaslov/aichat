# Survey AI — глубинное интервью через LLM

Веб-инструмент: задаёт респонденту один вопрос, отправляет ответ в LLM с ролью
«исследователь» и либо задаёт **один** уточняющий вопрос (если ответ поверхностный),
либо благодарит и завершает (если ответ подробный). Диалог можно продолжить.

- **Бэкенд:** FastAPI (отдаёт фронт и проксирует запрос в LLM).
- **Фронт:** один `index.html` (vanilla JS, без сборки).
- **LLM:** OpenRouter (OpenAI-совместимый API), бесплатная модель по умолчанию.

## Быстрый старт

### 1. Получить ключ
Бесплатный ключ — на https://openrouter.ai/keys (формат `sk-or-v1-...`).

### 2. Настроить окружение
```bash
cp .env.example .env
# открыть .env и вставить свой ключ в OPENROUTER_KEYS
```

### 3a. Запуск локально
```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload
```
Открыть http://localhost:8000

### 3b. Запуск в Docker
```bash
docker compose up --build
```
Открыть http://localhost:8000

## Тесты
```bash
pytest -q
```
Тесты полностью замокивают LLM — реальный ключ и сеть не нужны.

## Как устроено

| Файл          | Назначение                                                        |
|---------------|-------------------------------------------------------------------|
| `main.py`     | FastAPI: `GET /` (страница) и `POST /api/chat` (диалог → LLM).     |
| `llm.py`      | Клиент OpenRouter + системный промпт + перебор ключей.             |
| `index.html`  | Весь фронт: карточка, пузыри диалога, typing-эффект.               |
| `tests/`      | pytest на клиент LLM и HTTP-слой.                                  |

**Перебор моделей и ключей.** `MODEL` — список моделей через запятую, `OPENROUTER_KEYS` —
список ключей. Клиент перебирает их по порядку: если модель занята или ключ отклонён
(401/402/404/429 или сеть), берётся следующая комбинация. По умолчанию — только
бесплатные `:free` модели (ноль затрат). Чтобы поднять качество, поставьте платную модель
(`openai/gpt-4o-mini` и т.п.) **первой**, а бесплатные оставьте как fallback — при отсутствии
баланса запрос сам уйдёт на бесплатную. Подробности — в `.env.example`.

**История диалога** живёт в браузере и целиком отправляется на каждый запрос —
сервер не хранит состояние.

Обоснование системного промпта и решений по коду — в [COVER_LETTER.md](COVER_LETTER.md).
