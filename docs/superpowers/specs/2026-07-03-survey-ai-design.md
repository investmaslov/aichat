# Survey AI — дизайн-спека

Тестовое задание: веб-инструмент, который задаёт респонденту один вопрос и через LLM
решает — задать уточняющий вопрос или поблагодарить и завершить.

Репозиторий: `git@github.com:investmaslov/aichat.git`

## Цель

Пользователь отвечает на вопрос «Расскажите, как вы выбирали последний онлайн-курс?».
Ответ уходит на бэкенд → в LLM с ролью «исследователь». Модель либо задаёт ОДИН
уточняющий вопрос (если ответ поверхностный), либо благодарит и завершает (если ответ
подробный). Диалог можно продолжить: ответить на уточнение и получить финальную
благодарность.

## Стек

- **Бэкенд:** Python 3.12, FastAPI, uvicorn, httpx.
- **Фронт:** один файл `index.html` (inline CSS + vanilla JS). Отдаётся тем же сервером.
- **LLM:** OpenRouter (OpenAI-совместимый API), бесплатная модель по умолчанию.
- **Инфра:** Docker + docker-compose, GitHub Actions CI, pytest.

## Компоненты

### `llm.py` — клиент с перебором ключей
- `get_keys() -> list[str]` — парсит `OPENROUTER_KEYS` (через запятую), чистит пробелы/пустые.
- `async chat(messages: list[dict]) -> str` — подставляет системный промпт первым сообщением,
  перебирает ключи по очереди: на 401/402/429 или сетевой ошибке переходит к следующему ключу,
  возвращает первый успешный ответ. Если все ключи упали — бросает `LLMError` с понятным текстом.
- Модель берётся из `MODEL` (env), по умолчанию `meta-llama/llama-3.3-70b-instruct:free`.
- Зависит от: `httpx`, переменных окружения. Не зависит от FastAPI (тестируется изолированно).

### `main.py` — FastAPI
- `GET /` → отдаёт `index.html`.
- `POST /api/chat`, тело `{ "messages": [{"role","content"}, ...] }` (Pydantic-валидация,
  непустой список, непустой content) → зовёт `llm.chat` → `{ "reply": "..." }`.
  При `LLMError` → HTTP 502 с понятным сообщением.
- Зависит от: `llm.py`.

### `index.html` — фронт
- Держит историю диалога в JS-массиве.
- По «Отправить»: добавляет пузырь пользователя, шлёт всю историю на `/api/chat`,
  показывает индикатор «печатает…», затем печатает ответ typing-эффектом и добавляет в историю.
- Поле остаётся активным → диалог продолжается.
- Ошибка → красный пузырь, кнопка снова активна. Пустой ввод → кнопка неактивна.

## Системный промпт (ключевое решение)

```
Ты — вдумчивый UX-исследователь. Твоя задача — провести короткое глубинное интервью.

Респондент отвечает на вопрос: «Расскажите, как вы выбирали последний онлайн-курс?».

Оцени глубину ответа. Ответ ПОВЕРХНОСТНЫЙ, если в нём нет конкретики: не названы
критерии выбора, источники информации, сравнение вариантов, цена, отзывы или личный опыт.
Ответ ПОДРОБНЫЙ, если он раскрывает хотя бы 2–3 таких аспекта.

Если ответ поверхностный — задай РОВНО ОДИН короткий, тёплый уточняющий вопрос,
который поможет раскрыть мотивацию или критерии. Только один вопрос, без вступлений.

Если ответ подробный — искренне поблагодари и заверши интервью одной-двумя фразами.
Не задавай новых вопросов.

Отвечай на русском, дружелюбно и кратко. Не выходи за рамки темы онлайн-курсов.
```

**Почему так:** ТЗ задаёт лишь скелет («поверхностный → вопрос, подробный → благодарность»).
Проблема формулировки из ТЗ — модель не знает, что считать «поверхностным», и склонна
задавать несколько вопросов сразу. Поэтому в промпте: (1) явный операционный критерий
глубины через перечень аспектов, (2) жёсткое «РОВНО ОДИН вопрос», (3) фиксированный язык
и тон, (4) рамки темы, чтобы модель не уходила в сторону. Это делает поведение
предсказуемым и воспроизводимым — то, что и проверяет задание.

## Конфигурация (`.env.example`)

```
# Один или несколько ключей OpenRouter через запятую — берётся первый рабочий
OPENROUTER_KEYS=sk-or-v1-xxxx,sk-or-v1-yyyy
# Модель (любая доступная на OpenRouter)
MODEL=meta-llama/llama-3.3-70b-instruct:free
```

## Визуал (Apple-стиль, светлый премиум)

- Светлый воздушный фон, мягкий градиент розовый→белый→голубой, лёгкая анимация «дыхания».
- По центру стеклянная карточка (frosted glass, тонкая рамка, мягкая тень, крупный радиус).
- Крупный вопрос-заголовок, шрифт system-ui/Inter, много воздуха.
- Пузыри диалога: пользователь и ИИ визуально различаются, плавное появление.
- Индикатор «печатает…» (три точки) → typing-эффект текста ответа.
- Кнопка с мягким градиентом, hover/active/disabled-состояния, аккуратный focus-ring.
- Адаптивно под мобильные.

## Обработка ошибок

- Пустой ввод → кнопка disabled.
- Все ключи упали / сеть → HTTP 502 → красный пузырь на фронте, кнопка снова активна.
- Невалидное тело запроса → 422 (Pydantic).

## Тесты (pytest)

`tests/test_llm.py` (httpx замокан):
- парсинг `OPENROUTER_KEYS` (пробелы, пустые, один ключ);
- первый ключ рабочий → его ответ;
- первый ключ 401 → берётся второй, успех;
- все ключи падают → `LLMError`;
- системный промпт подставляется первым сообщением с ролью `system`.

`tests/test_api.py` (TestClient, `llm.chat` замокан):
- `POST /api/chat` успех → `{ "reply": ... }`;
- пустой `messages` → 422;
- `LLMError` → 502;
- `GET /` → 200, отдаёт HTML.

## Docker

- `Dockerfile`: python:3.12-slim, ставит `requirements.txt`, запускает
  `uvicorn main:app --host 0.0.0.0 --port 8000`.
- `docker-compose.yml`: проброс `8000:8000`, `env_file: .env`.
- Локальный запуск: `docker compose up` → `http://localhost:8000`.

## CI (`.github/workflows/ci.yml`)

На push и pull_request:
- setup Python 3.12, `pip install -r requirements.txt`;
- `pytest` (без реальных ключей — LLM замокан, секреты не нужны);
- сборка Docker-образа (`docker build`) как проверка, что образ собирается.

## Вне объёма (YAGNI)

- Никакой БД, авторизации, хранения сессий на сервере — история живёт в браузере.
- Нет жёсткого детектора «интервью завершено» — завершение проявляется естественно
  через текст модели; поле ввода всегда доступно.
- Нет стриминга токенов из LLM — typing-эффект рисуется на фронте по готовому ответу.
